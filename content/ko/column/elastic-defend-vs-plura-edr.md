📉 최근 보안 시장에서는 EDR, XDR, AI, 자율 대응 같은 표현이 함께 쓰이면서  
서로 다른 설계 철학의 제품이 마치 같은 방식으로 동작하는 것처럼 보일 때가 많습니다.

Elastic Defend와 PLURA-EDR도 이름만 보면 비슷해 보일 수 있습니다.

하지만 먼저 분명히 해야 할 점이 있습니다.

> **두 제품은 같은 EDR 범주에 놓일 수는 있어도,  
> 로그를 수집하고 위협을 해석하는 철학은 다릅니다.**

이 글의 핵심은 Elastic Defend를 깎아내리거나,  
반대로 PLURA-EDR을 과장하는 데 있지 않습니다.

오히려 반대로,  
**Elastic Defend는 어디에 강하고 어디까지를 기본 전제로 삼는지**,  
그리고  
**PLURA-EDR은 무엇을 더 기본적인 전제로 두고 설계되었는지**를  
구분해 보자는 데 있습니다.

즉, 질문은 이것입니다.

> **Elastic Defend와 PLURA-EDR 중 무엇이 무조건 더 낫냐?**  
> 가 아니라,  
> **두 제품은 무엇을 우선하도록 설계되었는가?**

---

## 1. 먼저 결론부터 말하면

Elastic Defend와 PLURA-EDR의 가장 큰 차이는  
**수집의 기본값과 운영 목적**에 있습니다.

Elastic 공식 문서에 따르면 Elastic Defend는  
위협 탐지와 예방을 위해 시스템 활동 데이터를 **선별적으로 수집**하며,  
저장 비용과 성능의 균형을 위해  
**모든 시스템 이벤트를 완전하게 캡처하도록 설계된 것은 아니다**라고 설명합니다.

또 데이터는 위협 탐지와 예방에 맞게  
집계되거나, 잘리거나, 중복 제거될 수 있다고 밝힙니다.

반면 PLURA 공식 문서는  
호스트보안(EDR)을 엔드포인트 수준의 다양한 시스템 로그와 보안 이벤트를  
**통합 수집·분석**하여 침해 탐지와 위협 행위 추적을 수행하는 기능으로 설명합니다.  
또 운영체제별 이벤트를 실시간 감시하고  
이상 징후나 보안 침해 시도를 탐지·차단할 수 있도록  
시각화된 로그를 제공한다고 설명합니다.

즉, 매우 단순하게 정리하면 이렇습니다.

1. **Elastic Defend는 보호 중심 EDR**
2. **PLURA-EDR은 기록·추적 중심 EDR**

따라서 둘 다 EDR이기는 하지만,  
운영자가 실제로 얻게 되는 **로그의 깊이와 맥락의 성격**은 다를 수 있습니다.

---

## 2. Elastic Defend는 무엇에 초점을 둔 제품인가?

Elastic Defend는 Elastic Security 안에서 동작하는  
엔드포인트 보호 및 위협 탐지 기능입니다.

Elastic 공식 문서상 이 제품은  
멀웨어 방어, 랜섬웨어 방어, 메모리 위협 방어, 악성 행위 방어 등  
보호 기능을 기본으로 제공하며,  
설정 화면에서 운영체제별 이벤트 수집 카테고리도 선택할 수 있습니다.

하지만 여기서 중요한 점은 따로 있습니다.

Elastic은 같은 공식 문서에서  
Elastic Defend가 시스템 활동을 **선별적으로 수집**하며,  
필요에 따라 이벤트가 **집계, 절단, 중복 제거**될 수 있다고 설명합니다.  
그리고 네트워크 포트 생성, 사용자 행위, 서비스 등록/변경 같은  
상세 Windows 이벤트가 필요하면  
**Custom Windows Event Logs integration** 같은 추가 수집기를 사용하라고 직접 안내합니다.

예를 들어 Elastic 문서는 다음과 같이 설명합니다.

- 모든 시스템 이벤트의 완전 캡처를 기본 목표로 하지 않음
- 로그인/로그아웃, 계정 생성/삭제/수정 같은 모든 사용자 이벤트를 다 담지 않음
- 서비스 생성/변경 이벤트도 행동 보호 엔진에 필요한 이벤트 중심
- 필요한 경우 Windows 보안 이벤트를 별도 integration으로 확장

이 말의 의미는 분명합니다.

> **Elastic Defend는 상세 로그를 빠짐없이 남기는 포렌식 수집기라기보다,  
> 탐지와 예방에 필요한 데이터를 효율적으로 수집하는 EDR에 가깝습니다.**

따라서 Elastic Defend의 강점은 다음에 가깝습니다.

- 엔드포인트 보호
- 악성 행위 탐지와 예방
- 저장 비용과 성능을 고려한 수집
- Elastic 생태계와의 확장성
- 필요한 경우 integration을 통한 보완

즉, Elastic Defend는  
**“기본적으로 다 남기는 제품”** 이라기보다  
**“기본적으로 잘 막기 위해 필요한 것을 남기는 제품”** 으로 이해하는 편이 더 정확합니다.

---

## 3. PLURA-EDR은 무엇에 초점을 둔 제품인가?

PLURA 공식 문서에서 호스트보안(EDR)은  
계정 탈취 공격, 원격 접속 시도, 권한 상승, 측면 이동 공격을 탐지·대응하며,  
엔드포인트 수준에서 발생하는 다양한 시스템 로그와 보안 이벤트를  
통합 수집·분석하여 침해 탐지와 위협 행위 추적을 수행하는 기능으로 설명됩니다.

특히 문서에서 더 눈에 띄는 부분은  
운영체제별 이벤트를 실시간으로 감시하고,  
Auditlog, Syslog, ADWS, Logcat 등 다양한 로그를 함께 본다는 점입니다.

즉, PLURA-EDR은 단순히  
“보호 결과만 보여주는 EDR”보다  
**행위의 흐름을 남기고 추적하는 EDR**에 더 가깝습니다.

또 PLURA 문서 전반에서는  
호스트보안과 마이터어택, 상관분석이 서로 연결되어 있습니다.

마이터어택 페이지에서는  
PLURA가 각 로그 이벤트를 MITRE ATT&CK 기반으로 분석하고,  
공격자가 어떤 단계에서 어떤 행동을 수행했는지 식별하며,  
탐지·차단 이벤트를 TID 단위로 분류해 제공한다고 설명합니다.

즉, PLURA-EDR의 핵심은 다음과 같습니다.

- 다양한 시스템 로그와 보안 이벤트의 통합 수집
- 실시간 감시와 행위 추적
- 계정 탈취, 원격 접속, 권한 상승, 측면 이동 탐지
- MITRE ATT&CK 기반 분류
- 침해 이후 근거와 흐름을 해석하기 쉬운 구조

따라서 PLURA-EDR은  
**“탐지 결과 중심 EDR”** 이라기보다  
**“상세 로그와 행위 추적 중심 EDR”** 으로 이해하는 편이 맞습니다.

---

## 4. 가장 중요한 차이: 로그 철학이 다릅니다

이 두 제품의 차이를 가장 정확하게 설명하는 방법은  
기능 목록을 길게 나열하는 것이 아니라,  
**로그를 대하는 철학이 다르다**고 보는 것입니다.

### 4.1 Elastic Defend의 철학
Elastic Defend는  
탐지와 예방에 필요한 데이터를 효율적으로 수집하고,  
저장 비용과 성능을 함께 고려합니다.  
그래서 공식 문서도  
완전한 시스템 이벤트 캡처를 기본 설계 목표로 두지 않았다고 설명합니다.

### 4.2 PLURA-EDR의 철학
PLURA-EDR은  
다양한 시스템 로그와 보안 이벤트를 실시간으로 통합 수집·분석하고,  
위협 행위를 추적하고,  
침해 시도를 시각화된 로그로 해석하도록 돕는 데 더 무게를 둡니다.

바로 이 차이 때문에  
같은 EDR로 분류하더라도  
운영자가 실제 현장에서 받게 되는 느낌은 달라질 수 있습니다.

---

## 5. “Elastic Defend는 고급 감사 정책을 못 쓰는가?”는 질문에 대하여

이 부분은 자주 오해가 생기는 지점입니다.

정확히 말하면,  
**Elastic Defend가 Windows 고급 감사 정책 자체를 못 쓰는 것은 아닙니다.**

다만 Elastic 공식 문서가 분명히 밝히듯이,  
Elastic Defend는 사용자 이벤트, 서비스 등록/변경, 특정 보안 이벤트 등에 대해  
완전 수집을 기본으로 하지 않으며,  
그런 상세 이벤트가 필요하면  
**Custom Windows Event Logs integration** 같은 추가 수집 구성을 활용하라고 안내합니다.

즉, 더 정확한 표현은 이렇습니다.

> **Elastic Defend 단독으로는 고급 감사 정책 기반 상세 이벤트를  
> 포렌식 수준으로 완전 수집하는 데 한계가 있고,  
> 상세 감사 로그 확보를 위해서는 추가 수집 구성이 필요합니다.**

이 차이는 매우 중요합니다.

왜냐하면 실무에서 필요한 질문은  
“수집이 되느냐 안 되느냐”보다  
**“기본 제품 설계가 어디까지를 기본값으로 삼는가”** 이기 때문입니다.

---

## 6. 두 제품을 표로 비교하면

| 항목 | Elastic Defend | PLURA-EDR |
|---|---|---|
| 기본 목적 | 위협 예방·탐지 중심 EDR | 상세 행위 기록·추적 중심 EDR |
| 로그 수집 철학 | 성능·저장 비용을 고려한 선별 수집 | 다양한 시스템 로그와 보안 이벤트의 통합 수집·분석 |
| 시스템 이벤트 처리 | 위협 탐지·예방에 필요한 수준으로 최적화 | 실시간 감시와 시각화된 로그 제공 강조 |
| 고급 감사 정책 상세 로그 | 기본 완전 수집 아님, 추가 integration 필요 가능 | 운영체제별 이벤트 기반 상세 추적 구조 강조 |
| 계정/원격접속/권한상승 추적 | 일부는 추가 보완 필요 | 문서상 핵심 탐지·대응 항목으로 제시 |
| 행위 흐름 해석 | 탐지 중심 | 위협 행위 추적 중심 |
| MITRE ATT&CK 연계 | Elastic Security 전체에서 활용 가능 | 로그 이벤트를 TID 단위로 분류하여 제공 |
| 적합한 환경 | Elastic 생태계 중심, 보호·예방 중심 운영 | 감사·증적·행위 추적이 중요한 운영 환경 |

이 비교의 핵심은  
Elastic이 부족하다는 뜻이 아닙니다.

오히려 반대로,  
Elastic Defend는 **보호 중심 EDR로서 목적이 분명한 제품**이고,  
PLURA-EDR은 **기록과 위협 행위 추적에 더 무게를 둔 EDR**이라는 뜻입니다.

---

## 7. 그래서 어떤 조직에 무엇이 더 맞는가?

### 7.1 Elastic Defend가 더 자연스러운 경우
다음과 같은 조직에는 Elastic Defend가 더 잘 맞을 수 있습니다.

- 이미 Elastic Stack을 폭넓게 사용하고 있는 경우
- 엔드포인트 위협 예방이 우선인 경우
- 로그 수집 범위를 운영 목적에 맞게 선별하고 싶은 경우
- 상세 Windows 보안 이벤트는 필요할 때 integration으로 보완하려는 경우

즉,  
**호스트 보호 중심 운영 + Elastic 확장성**이 중요하면  
Elastic Defend는 충분히 설득력 있는 선택입니다.

### 7.2 PLURA-EDR이 더 자연스러운 경우
반대로 다음과 같은 조직에는 PLURA-EDR이 더 직접적일 수 있습니다.

- 계정 탈취, 원격 접속, 권한 상승, 측면 이동 추적이 중요한 경우
- 운영체제 감사 로그와 보안 이벤트를 통합적으로 보고 싶은 경우
- 침해 이후 근거와 흐름을 더 자세히 확인하고 싶은 경우
- MITRE ATT&CK 기준으로 행위를 해석하고 싶은 경우
- 탐지뿐 아니라 기록과 추적 자체가 중요한 경우

즉,  
**상세 로그 기반의 근거 중심 운영**을 하려는 조직에는  
PLURA-EDR의 구조가 더 직접적으로 맞닿아 있습니다.

---

## 8. 진짜 질문은 “누가 더 좋으냐”가 아닙니다

실무에서 더 중요한 질문은 이것입니다.

> **우리 조직이 지금 필요한 것은  
> 예방 중심의 선별 수집 EDR인가?**  
> **아니면 기록과 행위 추적을 더 기본값으로 두는 EDR인가?**

Elastic Defend와 PLURA-EDR은  
이 질문에 대해 서로 다른 답을 내놓는 제품입니다.

Elastic Defend는  
**효율적인 보호와 예방**에 더 가깝고,  
PLURA-EDR은  
**상세 기록과 위협 행위 추적**에 더 가깝습니다.

따라서 제품 비교의 기준도  
“누가 더 많은 기능 이름을 갖고 있느냐”가 아니라,

> **무엇을 기본적으로 수집하는가**  
> **어디까지를 기본 제품 설계로 포함하는가**  
> **운영자가 침해 이후 어떤 근거와 맥락을 확인할 수 있는가**

여기에 맞춰져야 합니다.

---

## 9. 결론

Elastic Defend와 PLURA-EDR은  
같은 EDR 범주에 놓일 수 있지만,  
같은 기본 철학으로 설계된 제품은 아닙니다.

Elastic Defend는  
저장 비용과 성능을 고려해  
탐지·예방에 필요한 데이터를 선별적으로 수집하는  
**보호 중심 EDR**에 가깝습니다.  
상세 Windows 보안 이벤트나 포렌식 수준의 완전 수집이 필요하면  
추가 integration이 필요합니다.

반면 PLURA-EDR은  
다양한 시스템 로그와 보안 이벤트를 실시간으로 통합 수집·분석하고,  
계정 탈취, 원격 접속, 권한 상승, 측면 이동 같은 행위를 추적하며,  
MITRE ATT&CK 기반으로 위협 행위를 분류하는  
**기록·추적 중심 EDR**에 가깝습니다.

더 정확한 한 줄 결론은 이렇습니다.

> **Elastic Defend는 보호 중심 EDR이고,  
> PLURA-EDR은 기록과 행위 추적 중심 EDR입니다.**

제품 비교의 출발점이 되어야 합니다.
