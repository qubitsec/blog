## “눈에 띄지 않는 해킹 위협, 지금도 진행 중일 수 있습니다.”

해킹은 어느 날 갑자기 발생하지 않습니다.  
조용히, 깊숙이 파고들어 내부 시스템 어딘가에서 **눈에 띄지 않게 진행 중**일 가능성이 더 높습니다.

- 이미 해커가 **백도어를 설치해 두었을지도** 모릅니다.
- 지금 이 순간, 누군가가 데이터를 훔치고 있을지도 모릅니다.
- 그러나 문제는, 우리는 **그 사실조차 알기 어렵다**는 점입니다.

## “지금 당장 공격 중인가요?” 이 질문에 답할 수 있어야 합니다

보안의 출발점은 **상황 인식(Situational Awareness)** 입니다.  
그런데 많은 보안 시스템은 현재 무슨 일이 일어나고 있는지조차 **즉시 보여주지 못합니다.**

- 로그는 쌓이지만, **분석은 느립니다.**
- 감지가 되더라도, **대응은 수동입니다.**
- 탐지 범위가 제한적이어서, **지금 해킹이 진행 중인지 확인하려면 수시간 이상** 걸리기도 합니다.

이런 방식으로는 **APT 공격**, **파일리스 공격**, **백도어 설치 여부**를  
실시간에 가깝게 식별하기 어렵습니다.

즉, 문제는 탐지 장비가 없는 것이 아니라,  
**탐지 → 분석 → 대응이 하나의 흐름으로 이어지지 않는다**는 데 있습니다.

---

## PLURA-XDR은 왜 “1분 안에 확인”을 말할 수 있을까

PLURA-XDR이 말하는 “1분”은 단순한 마케팅 문구가 아니라,  
**수집 → 상관 분석 → 우선순위화 → 자동 대응**의 흐름을 짧게 만든 결과를 의미합니다.

핵심은 다음과 같습니다.

- **수집**: 웹, 시스템, 프로세스, 파일, 계정, 네트워크 로그를 실시간에 가깝게 수집
- **분석**: 개별 이벤트가 아니라 **연결된 행위 흐름**으로 해석
- **판단**: 단순 시그니처가 아니라 이상 징후와 문맥을 함께 평가
- **조치**: 사전에 정의된 정책에 따라 차단·격리·알림을 자동 실행
- **기록**: 이후 조사에 필요한 근거를 자동으로 남김

즉, “1분 안에 해킹 여부를 확인한다”는 말은  
단일 로그 한 줄을 보는 것이 아니라,  
**공격 징후를 사건 단위로 빠르게 묶어 보여주고 필요한 조치를 시작한다**는 뜻에 가깝습니다.

---

## 🔍 1분 내 자동 대응이 가능한 이유

### 1) 실시간 로그와 이벤트를 함께 봅니다
PLURA-XDR은 다음과 같은 데이터를 함께 봅니다.

- 프로세스 실행
- 파일 생성 및 변경
- 외부 연결
- 계정 사용 흔적
- 웹 요청 및 응답
- 시스템 이벤트와 감사 로그

중요한 것은 “많이 본다”가 아니라,  
이 데이터를 **같은 시간축과 같은 문맥으로 연결**한다는 점입니다.

예를 들어,

- 의심스러운 웹 요청이 발생하고
- 직후 비정상 프로세스가 실행되며
- 외부 연결이 이어지고
- 특정 계정 권한 사용이 증가했다면

이를 각각 따로 보는 것이 아니라  
**하나의 침해 흐름**으로 묶어 볼 수 있어야 합니다.

### 2) AI는 ‘보조 엔진’으로 동작합니다
PLURA-XDR의 AI는 단순히 “이상하다”고 말하는 용도가 아니라,  
여러 이벤트 간의 연관성을 빠르게 정리하고  
우선순위를 높이는 보조 역할을 합니다.

즉,

- 정상처럼 보이는 이벤트를 문맥으로 재해석하고
- 반복되는 노이즈를 줄이며
- 사람이 바로 이해할 수 있는 사건 형태로 정리해

보안 담당자가 더 빨리 판단할 수 있도록 돕습니다.

### 3) 자동 대응 규칙이 이미 준비돼 있습니다
탐지 후 사람이 하나씩 승인하는 구조라면  
1분 내 대응은 현실적으로 어렵습니다.

그래서 중요한 것은 **사전에 정의된 대응 시나리오**입니다.

예를 들어,

- 특정 공격 패턴이 확인되면 해당 IP 차단
- 비정상 프로세스 체인이 탐지되면 격리
- 백도어성 외부 연결이 발생하면 통신 차단 및 알림

처럼, 조건이 충족되면  
즉시 조치가 이어지도록 설계해야 합니다.

즉, “1분”은 AI 하나의 힘이 아니라,  
**실시간 수집 + 상관 분석 + 사전 정의된 자동 대응**이 함께 작동할 때 가능한 시간입니다.

## 💡 실제 도입 사례

### 사례 A: SQL 인젝션 공격 → 자동 차단

#### A 기업의 고민
- 오랜 시간 SQL 인젝션에 대비해 왔지만, 공격 패턴은 계속 우회됨
- 국내외 IP를 가리지 않고 공격이 반복됨
- 특히 주말·휴일 새벽처럼 대응이 취약한 시간대를 노림
- 외부 관제 서비스를 이용해도 **즉시 차단**까지는 어려움
- 자체 24×365 대응팀 운영은 비용과 인력 측면에서 부담이 큼

#### 실제로 본 것은 무엇인가
PLURA-XDR은 단순히 “SQL Injection 문자열” 하나만 본 것이 아니라,

- 웹 요청 본문과 파라미터 이상 징후
- 동일 출발지의 반복 시도 패턴
- 비정상 요청 흐름
- 차단 기준을 넘는 행위 빈도

를 함께 평가해  
단일 요청이 아니라 **공격 시도 흐름**으로 해석합니다.

#### 대응 결과
- 의심 행위가 기준을 넘으면 즉시 IP 접속 차단
- 반복 공격과 변칙 패턴을 자동으로 억제
- 이후 관련 로그와 근거를 자동 보관

#### 도입 효과
과거에는 “공격 발생 후 인지 → 수동 확인 → 수동 차단” 흐름이었지만,  
도입 후에는 **보안 담당자가 출근하기 전**에  
이미 차단과 기록이 완료되는 구조로 바뀌었습니다.

핵심은 “한 번에 완벽히 다 안다”가 아니라,  
**첫 번째 신호에서 피해를 줄이는 방향으로 먼저 반응할 수 있게 된 것**입니다.

---

### 사례 B: APT(지능형지속위협) → 행위 기반 탐지

#### B 기업의 고민
- 시스템 CPU 과부하와 반복 장애가 발생
- 원인을 찾아도 정확한 근본 원인이 보이지 않음
- 여러 백신을 사용해도 탐지 실패
- 시스템 재설치는 부담이 커 현실적으로 어려움
- 주말마다 장애가 반복돼 담당자가 긴급 출근해야 했음

#### 실제로 본 것은 무엇인가
이 사례에서 중요한 것은 “악성코드 파일 이름”이 아니라  
**행위 패턴**이었습니다.

예를 들어,

- PowerShell 같은 스크립트 기반 실행
- 비정상 프로세스 연쇄
- 지속성 확보 시도
- 외부 연결 또는 자원 과다 사용
- 시간대와 행위 반복성

같은 신호를 종합해 보면  
단순 장애가 아니라 **공격 또는 침해 행위의 흔적**으로 읽힐 수 있습니다.

PLURA-XDR은 이런 행위를 **MITRE ATT&CK 관점**으로 해석하고,  
관련 흐름을 **6W1H 형태**로 정리해  
담당자가 원인과 우선순위를 빠르게 이해할 수 있게 돕습니다.

#### 도입 효과
이전에는 공격 징후를 보더라도  
정확한 의미를 파악하고 대응하기까지 시간이 오래 걸렸지만,  
도입 후에는 **출근 전에 이미 차단·격리·기록**이 진행되어  
주말 장애와 보안 공백을 크게 줄일 수 있었습니다.

핵심은 “악성 파일을 찾았다”보다,  
**이상 행위를 사건 단위로 묶어 빠르게 대응한 것**입니다.

## 기존 방식이 느린 이유는 “장비가 없어서”가 아닙니다

많은 조직이 보안 솔루션을 이미 갖고 있습니다.  
그런데도 느린 이유는 보통 다음과 같습니다.

- 로그는 있지만 서로 연결되지 않음
- 탐지는 되지만 사건으로 묶이지 않음
- 분석은 되지만 대응이 사람 손을 기다림
- 대응은 가능하지만 근거가 흩어져 있음

즉, 문제는 솔루션 수가 아니라  
**운영 구조가 실시간 대응에 맞게 설계되어 있지 않다**는 데 있습니다.

PLURA-XDR의 차별점은 바로 여기에 있습니다.  
탐지 기능 하나를 더 붙이는 것이 아니라,  
**현재 진행 중인 공격을 더 빨리 사건으로 인식하고 대응까지 이어지게 만드는 구조**를 제공하는 데 있습니다.

---

## 지금은 “탐지”보다 “즉시 대응”이 중요한 시대입니다

해커는 자동화된 공격 도구로 움직이는데,  
우리는 아직도 사람이 로그를 하나씩 보며  
수동으로 조치하고 있지 않은가를 돌아봐야 합니다.

PLURA-XDR은 사람이 인지하기 전에  
이미 대응이 시작될 수 있도록 설계된 플랫폼입니다.

예를 들어,

- 백도어 설치 징후 탐지
- 정체불명의 외부 연결 차단
- 정상처럼 위장한 의심 프로세스 추적
- 웹 공격과 서버 행위의 연관 분석

이런 흐름을 사람이 개입하기 전에  
먼저 식별하고 필요한 차단을 시작할 수 있어야 합니다.

---

## 기억해야 할 핵심은 이것입니다

해킹은 갑자기 터지는 것이 아니라,  
**조용히 잠식되다가 어느 순간 폭발적으로 드러나는 사건**에 가깝습니다.

그래서 가장 중요한 질문은 이것입니다.

> **지금 무슨 일이 일어나고 있는가?**

그리고 그 다음 질문은 이것이어야 합니다.

> **그 사실을 언제, 어떤 근거로, 얼마나 빨리 알 수 있는가?**

PLURA-XDR이 말하는 1분의 의미는  
단순히 빠르다는 것이 아닙니다.  
**공격이 진행 중일 때, 아직 늦지 않았다는 뜻**입니다.
