## 🛡️ 웹방화벽(WAF) SaaS 선택 체크리스트

웹방화벽은 더 이상 단순한 **HTTP 요청 차단 장비**가 아닙니다.  

현대 공격은  
- 크리덴셜 스터핑  
- JWT 키 탈취  
- 웹쉘 업로드  
- 데이터 유출  
과 같이 **애플리케이션 + 계정 + 서버 내부까지 확장**됩니다.

따라서 WAF 선택의 기준은  
**차단 여부가 아니라 “탐지 + 분석 + 대응 가능성”**이어야 합니다.

특히 SaaS 기반 WAF는  
**전수 로그 수집과 실시간 분석 능력**이 핵심입니다.

---

## ✅ 필수 체크리스트 (핵심 10)

### 1. 공격 “차단”이 아닌 “탐지·분석·대응” 중심인가
- 단순 룰 기반 차단이 아닌  
- 공격 행위 전체를 분석하고 대응 가능한 구조인지

---

### 2. 요청 본문(Post-body)까지 수집·분석 가능한가
- Access Log 수준이 아닌  
- **HTTP Request Body 포함 전체 로그 분석 여부**
- SQLi, 웹쉘, API abuse 탐지 가능 여부

👉 핵심: **보이지 않으면 탐지도 없다**

---

### 3. 크리덴셜 스터핑 탐지 가능 여부
- 로그인 시도 패턴 분석 (IP / 계정 / 실패율 / 분산 공격)
- 정상 사용자와 공격자 구분 가능 여부
- 계정 탈취 시나리오 대응 여부

---

### 4. JWT 키 유출 기반 공격 탐지 가능 여부
- JWT 토큰 위변조 탐지
- 비정상 토큰 사용 패턴 분석
- **쿠팡 사례와 같은 인증 우회 공격 탐지 가능 여부**

---

### 5. 제로데이 공격 대응 구조
- 시그니처 기반이 아닌  
- **행위 기반 탐지(Behavior-based Detection)** 지원 여부
- 신규 공격도 탐지 가능한지

---

### 6. 데이터 유출(Exfiltration) 탐지 가능 여부
- 대량 응답 데이터 전송 탐지
- 특정 패턴 데이터 외부 전송 감지
- 내부 API 오용 탐지

---

### 7. 웹쉘 탐지 및 대응 (EDR 기능 포함 여부)
- 단순 웹 트래픽이 아닌  
- 서버 내부 행위까지 탐지 가능한지

필수 항목:
- 파일 생성/변경 감시
- 프로세스 실행 추적
- 웹쉘 실행 패턴 탐지

👉 **WAF + EDR 통합 여부는 필수**

---

### 8. 전수 로그 분석(Full Logging) 가능 여부
- 샘플링이 아닌 **100% 전체 로그 수집**
- 저장 + 재분석 가능 여부

👉 핵심 논리:
> 제로데이는 “지금 탐지”가 아니라  
> **“나중에라도 반드시 탐지”할 수 있어야 한다**

---

### 9. 실시간 탐지 및 대응 자동화
- 공격 발생 즉시 탐지 가능 여부
- 자동 차단 / 세션 종료 / 계정 보호 기능
- 알림 및 대응 연계

---

### 10. 통합 분석 플랫폼(XDR 연계)
- WAF 단독이 아닌  
- 서버, 계정, 네트워크와 연계 분석 가능 여부
- 공격 흐름 전체 추적 가능 여부

---

## ⚠️ SaaS WAF에서 반드시 걸러야 할 제품

### ❌ 1. 룰 기반 차단 중심 제품
- OWASP TOP 10만 대응
- 알려진 공격만 차단

👉 현실: **대부분 공격은 우회됨**

---

### ❌ 2. Access Log 기반 분석 제품
- 요청 본문 미수집
- 공격 payload 확인 불가

👉 현실: **탐지 불가능**

---

### ❌ 3. 샘플링 로그 구조
- 일부 로그만 저장
- 전체 분석 불가

👉 현실: **제로데이 대응 불가**

---

### ❌ 4. 서버 내부 행위 미가시화
- 웹쉘 탐지 불가
- 파일/프로세스 추적 불가

👉 현실: **침해 이후 무방비**

---

### ❌ 5. 단일 장비/단일 기능 제품
- WAF만 존재
- EDR / XDR 미연계

👉 현실: **공격 흐름 분석 불가능**

---

## 📊 핵심 평가 기준 요약

| 항목 | 필수 여부 | 핵심 포인트 |
|------|----------|------------|
| 요청 본문 로그 분석 | 필수 | 공격 payload 가시성 |
| 크리덴셜 스터핑 탐지 | 필수 | 계정 탈취 대응 |
| JWT 공격 탐지 | 필수 | 인증 우회 방지 |
| 제로데이 대응 | 필수 | 행위 기반 분석 |
| 데이터 유출 탐지 | 필수 | 내부 정보 보호 |
| 웹쉘 탐지 (EDR) | 필수 | 서버 침해 대응 |
| 전수 로그 분석 | 필수 | 사후 탐지 가능성 |
| 실시간 대응 | 필수 | 공격 차단 |
| XDR 연계 | 필수 | 전체 공격 흐름 분석 |

## 📌 결론

웹방화벽 선택의 기준은 명확합니다.

> **“모든 데이터를 기록하고,  
> 모든 공격을 분석할 수 있는가?”**

이 질문에 YES가 아니라면  
그 제품은 **보안 장비가 아니라 필터에 불과합니다.**

---

### 최종 메시지

보안은 차단이 아니라  
**기록 → 분석 → 대응의 문제입니다.**

그리고 그 출발점은 항상 동일합니다.

> **“전수 로그 수집”**

이 기준 없이  
제로데이를 말하는 것은  
기술이 아니라 환상입니다.
